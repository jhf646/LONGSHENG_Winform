# 上海隆深库存管理系统 - 部署手册

---

## 一、系统概述

### 1.1 系统组成

| 组件 | 技术栈 | 说明 |
|------|--------|------|
| **后端 API** | ASP.NET Core 10.0 + SQL Server | RESTful 服务，端口 5000 |
| **Web 前端** | HTML/CSS/JavaScript | 浏览器访问，无需安装 |
| **WinForms 客户端** | .NET 9.0 WinForms | Windows 桌面应用（可选） |
| **数据库** | SQL Server (LocalDB / Express / Standard) | 数据持久化 |
| **PLC 通信** | Modbus TCP (NModbus) | 汇川 Easy522 堆垛机 |

### 1.2 运行环境要求

| 项目 | 最低要求 | 推荐 |
|------|---------|------|
| 操作系统 | Windows 10 / Windows Server 2016+ | Windows 11 / Server 2022 |
| .NET SDK | .NET 10.0 SDK | .NET 10.0 SDK |
| SQL Server | SQL Server 2019+ (任意版本) | SQL Server 2022 Express |
| 内存 | 4 GB | 8 GB |
| CPU | 双核 2.0 GHz | 四核 2.5 GHz+ |
| 硬盘 | 1 GB 可用空间 | 10 GB+（含数据库增长） |
| 浏览器 | Chrome 90+ / Edge 90+ | Chrome 最新版 |

## 三、数据库配置

### 3.1 数据库自动创建

系统首次启动时会**自动创建**数据库 `LongShenStorage` 和所有表，**无需手动执行 SQL 脚本**。

自动创建的表清单：

| 表名 | 用途 |
|------|------|
| `AlertSettings` | 库存预警阈值 |
| `StorageSlots` | 货位定义（2排×4列×8层=64个） |
| `WorkpieceRecords` | 在库工件记录 |
| `LedgerEntries` | 进出台账 |
| `Users` | 用户账号 |
| `DropdownOptions` | 下拉选项持久化 |
| `RolePermissions` | 角色权限配置 |

### 3.2 手动创建（可选）

如需手动初始化数据库，可执行以下 SQL：

```sql
-- 创建数据库
IF NOT EXISTS (SELECT name FROM sys.databases WHERE name = N'LongShenStorage')
BEGIN
    CREATE DATABASE [LongShenStorage];
END
GO

USE [LongShenStorage];
GO

-- AlertSettings
IF NOT EXISTS (SELECT * FROM sysobjects WHERE name='AlertSettings' AND xtype='U')
CREATE TABLE AlertSettings (
    Id INT PRIMARY KEY DEFAULT 1 CHECK (Id = 1),
    MinThreshold INT NOT NULL DEFAULT 2,
    MaxThreshold INT NOT NULL DEFAULT 18
);

-- StorageSlots
IF NOT EXISTS (SELECT * FROM sysobjects WHERE name='StorageSlots' AND xtype='U')
CREATE TABLE StorageSlots (
    SlotCode NVARCHAR(50) PRIMARY KEY,
    IsOccupied BIT NOT NULL DEFAULT 0,
    WorkpieceId UNIQUEIDENTIFIER NULL,
    Zone NVARCHAR(50) NOT NULL DEFAULT '',
    RowNumber INT NOT NULL,
    ColumnNumber INT NOT NULL,
    LevelNumber INT NOT NULL
);

-- WorkpieceRecords
IF NOT EXISTS (SELECT * FROM sysobjects WHERE name='WorkpieceRecords' AND xtype='U')
CREATE TABLE WorkpieceRecords (
    Id UNIQUEIDENTIFIER PRIMARY KEY,
    InboundTime DATETIME2 NOT NULL DEFAULT GETDATE(),
    SlotCode NVARCHAR(50) NOT NULL,
    LastOperator NVARCHAR(100) NOT NULL DEFAULT '',
    LastUpdated DATETIME2 NOT NULL DEFAULT GETDATE(),
    Notes NVARCHAR(500) NOT NULL DEFAULT '',
    PalletNumber NVARCHAR(50) NOT NULL DEFAULT '',
    ToolingNumber NVARCHAR(200) NOT NULL DEFAULT '',
    ProjectNumber NVARCHAR(200) NOT NULL DEFAULT '',
    ModelType NVARCHAR(200) NOT NULL DEFAULT '',
    WorkOrder NVARCHAR(200) NOT NULL DEFAULT '',
    CellNumber NVARCHAR(200) NOT NULL DEFAULT '',
    ComponentSections INT NOT NULL DEFAULT 1,
    CustomerName NVARCHAR(200) NOT NULL DEFAULT ''
);

-- LedgerEntries
IF NOT EXISTS (SELECT * FROM sysobjects WHERE name='LedgerEntries' AND xtype='U')
CREATE TABLE LedgerEntries (
    Id UNIQUEIDENTIFIER PRIMARY KEY,
    Type INT NOT NULL,
    Timestamp DATETIME2 NOT NULL DEFAULT GETDATE(),
    OperatorName NVARCHAR(100) NOT NULL DEFAULT '',
    SlotCode NVARCHAR(50) NOT NULL DEFAULT '',
    ActionDescription NVARCHAR(500) NOT NULL DEFAULT '',
    PalletNumber NVARCHAR(50) NOT NULL DEFAULT '',
    ToolingNumber NVARCHAR(200) NOT NULL DEFAULT '',
    ProjectNumber NVARCHAR(200) NOT NULL DEFAULT '',
    ModelType NVARCHAR(200) NOT NULL DEFAULT '',
    WorkOrder NVARCHAR(200) NOT NULL DEFAULT '',
    CellNumber NVARCHAR(200) NOT NULL DEFAULT '',
    ComponentSections INT NOT NULL DEFAULT 0,
    CustomerName NVARCHAR(200) NOT NULL DEFAULT ''
);

-- Users
IF NOT EXISTS (SELECT * FROM sysobjects WHERE name='Users' AND xtype='U')
CREATE TABLE Users (
    Id UNIQUEIDENTIFIER PRIMARY KEY,
    Username NVARCHAR(100) NOT NULL UNIQUE,
    PasswordHash NVARCHAR(500) NOT NULL,
    Role INT NOT NULL DEFAULT 1,
    DisplayName NVARCHAR(100) NOT NULL DEFAULT '',
    IsActive BIT NOT NULL DEFAULT 1,
    CreatedAt DATETIME2 NOT NULL DEFAULT GETDATE()
);

-- DropdownOptions
IF NOT EXISTS (SELECT * FROM sysobjects WHERE name='DropdownOptions' AND xtype='U')
CREATE TABLE DropdownOptions (
    Category NVARCHAR(50) NOT NULL,
    Value NVARCHAR(200) NOT NULL,
    PRIMARY KEY (Category, Value)
);

-- RolePermissions
IF NOT EXISTS (SELECT * FROM sysobjects WHERE name='RolePermissions' AND xtype='U')
CREATE TABLE RolePermissions (
    RoleName NVARCHAR(50) NOT NULL,
    PageId NVARCHAR(50) NOT NULL,
    PRIMARY KEY (RoleName, PageId)
);
```

### 3.3 插入默认数据

首次启动时系统会自动插入以下数据：

**默认用户：**

| 用户名 | 密码 | 角色 | 权限 |
|--------|------|------|------|
| `admin` | `admin123` | 管理员 | 全部功能（含用户管理、角色权限） |
| `operator` | `123456` | 操作员 | 入库、出库、设备调用等操作功能 |

## 四、部署步骤

### 4.1 标准部署（推荐 - 单机部署）

#### 步骤 1：配置数据库连接

编辑 `LongShenStorageApi/appsettings.json`：

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=localhost;Database=LongShenStorage;User Id=sa;Password=123456;TrustServerCertificate=True;"
  }
}
```

**常用连接字符串示例：**

| 环境 | 连接字符串 |
|------|-----------|
| LocalDB (开发) | `Server=(LocalDB)\MSSQLLocalDB;Database=LongShenStorage;Trusted_Connection=True;TrustServerCertificate=True;` |
| SQL Server Express (本机) | `Server=localhost;Database=LongShenStorage;User Id=sa;Password=123456;TrustServerCertificate=True;` |
| 远程服务器 | `Server=192.168.1.100,1433;Database=LongShenStorage;User Id=sa;Password=your_password;TrustServerCertificate=True;` |
| Windows 集成认证 | `Server=localhost;Database=LongShenStorage;Trusted_Connection=True;TrustServerCertificate=True;` |

#### 步骤 2：启动 API 服务

```bash
# 开发模式（热重载）
cd LongShenStorageApi
dotnet run

# 生产模式（发布后运行）
dotnet publish -c Release -o ./publish
cd publish
dotnet LongShenStorageApi.dll

# 指定端口
dotnet run --urls "http://0.0.0.0:5000"
```

#### 步骤 3：验证启动

```bash
# 检查 API 是否正常运行
curl http://localhost:5000/api/appstate/dashboard

# 正常返回 JSON 数据即部署成功
```

#### 步骤 4：打开前端

直接用浏览器打开 `LongShenStorageWeb/index.html`

> ⚠️ 前端需要 API 服务运行在 `http://localhost:5000`。如需修改 API 地址，编辑 `js/app.js` 中的 `API_BASE` 变量。

### 4.2 Windows 服务部署（生产环境）

#### 步骤 1：发布 API

```bash
cd LongShenStorageApi
dotnet publish -c Release -o C:\LongShenStorage\Api
```

#### 步骤 2：创建 Windows 服务

使用 `sc` 命令创建服务：

```powershell
# 以管理员身份运行 PowerShell

# 创建服务
sc create "LongShenStorageApi" binPath="C:\LongShenStorage\Api\LongShenStorageApi.exe --urls http://0.0.0.0:5000" start=auto

# 设置服务描述
sc description "LongShenStorageApi" "上海隆深库存管理系统后端API服务"

# 启动服务
sc start "LongShenStorageApi"

# 查看状态
sc query "LongShenStorageApi"
```

#### 步骤 3：配置 IIS 反向代理（可选）

如需通过 IIS 80 端口访问，配置反向代理：

```xml
<!-- 需要安装 ARR (Application Request Routing) 模块 -->
<system.webServer>
  <rewrite>
    <rules>
      <rule name="ReverseProxy" stopProcessing="true">
        <match url="(.*)" />
        <action type="Rewrite" url="http://localhost:5000/{R:1}" />
      </rule>
    </rules>
  </rewrite>
</system.webServer>
```

#### 步骤 4：前端部署

将 `LongShenStorageWeb` 文件夹部署到 IIS 或 Nginx：

```powershell
# 复制前端文件到 IIS 目录
Copy-Item -Path "LongShenStorageWeb\*" -Destination "C:\inetpub\wwwroot\wms" -Recurse -Force
```

### 6.3 查看数据量

```sql
SELECT
    t.NAME AS 表名,
    p.rows AS 行数,
    SUM(a.total_pages) * 8 / 1024 AS 占用空间MB
FROM sys.tables t
JOIN sys.indexes i ON t.object_id = i.object_id
JOIN sys.partitions p ON i.object_id = p.object_id AND i.index_id = p.index_id
JOIN sys.allocation_units a ON p.partition_id = a.container_id
WHERE t.is_ms_shipped = 0
GROUP BY t.NAME, p.rows
ORDER BY 占用空间MB DESC;
```

## 七、常见问题

### Q1: API 启动报错 "无法连接到数据库"

```
System.Data.SqlClient.SqlException: 无法打开登录所请求的数据库
```

**解决方法：**
1. 确认 SQL Server 服务正在运行
2. 确认连接字符串中的用户名密码正确
3. 检查防火墙是否允许 1433 端口
4. 首次启动会自动创建数据库，需确保账号有 `CREATE DATABASE` 权限

### Q2: 前端打不开 / 白屏

**解决方法：**
1. 确认 API 服务已启动（`http://localhost:5000` 可访问）
2. 确认前端文件路径正确
3. 打开浏览器开发者工具（F12）→ Console 查看具体错误
4. 检查 `app.js` 中的 `API_BASE` 地址是否正确

### Q3: 数据库自动创建失败

**解决方法：**
1. 手动执行 SQL 脚本创建数据库和表
2. 确认当前用户有建库权限
3. 使用 `sa` 账号连接

### Q4: 端口 5000 被占用

```bash
# 修改端口
# 方式一：命令行指定
dotnet run --urls "http://0.0.0.0:5001"

# 方式二：修改 appsettings.json
"Urls": "http://0.0.0.0:5001"

# 同时修改前端 js/app.js 中的 API_BASE
const API_BASE = 'http://localhost:5001/api';
```

### Q5: JWT 登录失败 "用户名或密码错误"

**解决方法：**
1. 确认用户已创建（首次启动自动创建 admin/operator/viewer）
2. 如果用户表为空，重启 API 自动插入默认用户
3. 管理员可通过 API 创建新用户：`POST /api/auth/create-user`

## 八、安全建议

| 项目 | 建议 |
|------|------|
| **JWT密钥** | 修改 `Jwt:Key` 为至少 32 位随机字符串 |
| **默认密码** | 首次登录后立即修改 3 个默认账号的密码 |
| **数据库密码** | 不要使用弱密码 `123456` |
| **网络隔离** | API 服务不要暴露在公网，仅内网访问 |
| **HTTPS** | 生产环境建议配置 HTTPS |
| **备份** | 每天定时备份数据库 |
| **日志** | 定期检查 API 日志，关注异常登录尝试 |
