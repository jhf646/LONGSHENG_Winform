## 二、两种运行模式

### 模式一：模拟器模式（默认，无需硬件）

适合开发测试、演示、培训。

**配置：** `appsettings.json`
```json
"ModbusTcp": {
    "UseSimulator": true
}
```

**特点：**
- 无需连接真实PLC
- 内置模拟器自动响应读写请求
- 模拟 X/Y/Z 轴位置渐变运动
- 发送指令后模拟堆垛机从 A 点移动到 B 点

### 模式二：真实PLC模式

连接现场汇川 Easy522 堆垛机。

**配置：** `appsettings.json`
```json
"ModbusTcp": {
    "Host": "192.168.1.100",    // ← 改成实际PLC的IP地址
    "Port": 502,                 // Modbus TCP 默认端口
    "SlaveId": 1,                // 从站地址(站号)
    "UseSimulator": false        // ← 关闭模拟器
}
```

**接线要求：**
- 上位机与 PLC 通过以太网连接
- 确保 IP 在同一网段（例如 PLC=192.168.1.100，PC=192.168.1.x）
- 防火墙允许 TCP 502 端口通信

## 三、汇川 Easy522 PLC 设置

### 3.1 PLC 侧参数配置（使用汇川 AutoShop 软件）

1. 打开 AutoShop → 新建工程 → 选择 Easy522 CPU
2. 配置以太网参数：
   - **IP地址：** 根据现场网络设定（例如 `192.168.1.100`）
   - **子网掩码：** `255.255.255.0`
   - **网关：** 按需设定
3. 开启 Modbus TCP 服务器：
   - 在"通信配置"中启用 **Modbus TCP Server**
   - 端口号：`502`（默认）
   - 从站地址：`1`
4. D 区寄存器作为 **保持寄存器**（Holding Register）映射到 Modbus 地址
   - 汇川 D 区地址 = Modbus 地址（直接映射）
   - 例如：`D100` = 地址 `100`（功能码 03 读取 / 06 写入）
5. 下载程序到 PLC，重启生效

### 3.2 验证连接

在 PC 上用 Modbus Poll 测试：
1. 打开 Modbus Poll
2. 设置：`192.168.1.100:502`，站号 `1`
3. 功能码 `03` 读取保持寄存器
4. 地址 `100`，长度 `20`
5. 如果能读到数据，说明连接成功

## 四、D区寄存器协议定义

### 4.1 协议规则

| 项目 | 说明 |
|------|------|
| 数据类型 | Word（16位无符号整数），范围 0~65535 |
| 功能码读取 | 03（Read Holding Registers） |
| 功能码写入 | 06（Write Single Register） |
| **地址映射** | D区地址 = Modbus 地址（**注意：** 汇川 D100 = Modbus 地址 100） |
| **禁止使用 0 值** | 0 只作为判断是否读取成功的标志，建议数值 1 开始使用 |
| 数据进制 | 10 进制（界面显示和操作均为 10 进制） |

> ⚠️ **重要：** Modbus 地址与 PLC D 区地址的对应关系以实际 PLC 型号为准。
> 汇川 Easy 系列：`D100` 对应 Modbus 保持寄存器地址 `100`（功能码 03/06）。
> 部分其他品牌 PLC 可能需要地址偏移（如 `D100` = Modbus `400101`），请参考 PLC 手册。

### 4.2 读取寄存器（上位机 ← 堆垛机）

上位机通过功能码 **03** 读取以下寄存器，监控设备状态。

| D地址 | 名称 | 取值范围 | 说明 |
|-------|------|---------|------|
| D100 | 立库状态 | 1~4 | **1:** 空闲中 **2:** 运行中 **3:** 故障中 **4:** 暂停中 |
| D101 | 故障异常代码 | 0~N | **0:** 正常，其他为故障代码 |
| D102 | 运行模式 | 1~2 | **1:** 手动 **2:** 自动（自动才接受指令） |
| D103 | 动作步骤 | 0~9 | 当前执行到的步骤：**1:** XY定位到A点 **2:** Z轴在A点伸出 **3:** Y轴提升 **4:** Z收回 **5:** XY到B点 **6:** Y提升 **7:** Z在B伸出 **8:** Y下降 **9:** Z在B收回 |
| D104 | A→B任务完成 | 1~2 | **1:** 动作完成 **2:** 执行中 |
| D105 | 托盘转移状态 | 1~3 | **1:** 尚未取到托盘 **2:** 已转移到堆垛车 **3:** 已到目标位置 |
| D106 | 备用 | - | 预留 |
| D107 | X轴实时位置 | 0~N | X轴（列方向）当前位置，单位 mm |
| D108 | Y轴实时位置 | 0~N | Y轴（层方向）当前位置，单位 mm |
| D109 | Z轴深库位置 | 0~N | Z轴深库伸叉位置，单位 mm |
| D110 | Z轴浅库位置 | 0~N | Z轴浅库伸叉位置，单位 mm |
| D111 | 是否可以移库 | 1~2 | **1:** 可以 **2:** 不可以 |
| D112 | 左侧入库口可入库 | 1~2 | **1:** 可以 **2:** 不可以 |
| D113 | 左侧出库口可出库 | 1~2 | **1:** 可以 **2:** 不可以 |
| D114 | 右侧入库口可入库 | 1~2 | **1:** 可以 **2:** 不可以 |
| D115 | 右侧出库口可出库 | 1~2 | **1:** 可以 **2:** 不可以 |
| D116 | A→B动作是否已完成 | 0~1 | **0:** 未完成 **1:** 已完成 |
| D117 | 堆垛车当前位置 | 0~N | 101+ 车位号 |

### 4.3 写入寄存器（上位机 → 堆垛机）

上位机通过功能码 **06** 写入以下寄存器，下发指令。

> **写入流程：** 先写入动作参数（地址 101~112），最后写入 **D103=1** 触发执行。

| D地址 | 名称 | 取值范围 | 说明 |
|-------|------|---------|------|
| D101 | 设备序号 | 1~N | 堆垛机设备编号 |
| D102 | 序列1 | 1~N | 动作序列号，每次递增 |
| **D103** | **动作标志位** | **0~1** | **写入 1 触发执行！**（必须先填好其他参数） |
| D104 | A:排 | 11~12 | **11:** 入库位置 **12:** 出库位置 |
| D105 | A:列 | 1~4 | 取货位列号 |
| D106 | A:层 | 1~8 | 取货位层号 |
| D107 | B:排 | 1~2 | 放货位排号 |
| D108 | B:列 | 1~4 | 放货位列号 |
| D109 | B:层 | 1~8 | 放货位层号 |
| D110 | 附加参数1 | 按需 | 扩展功能参数 |
| D111 | 附加参数2 | 按需 | 扩展功能参数 |
| D112 | 动作类型 | 1~5 | **1:** 默认无动作 **2:** 出库 **3:** 入库 **4:** 移库 **5:** 直接出 |
| D113 | 出库后续动作 | 1,11~1n | **1:** 默认无动作 **11:** 线体1号位 **12:** 线体2号位 |
| D114 | 起始地址 | 0~N | 起始地址参数 |
| D115 | 目标地址 | 0~N | 目标地址参数 |
| D116 | 其他参数 | 0~N | 预留扩展 |

### 4.4 指令发送示例

**例：从 A(11排,1列,1层) 取货到 B(2排,4列,8层) 入库**

| 步骤 | D地址 | 写入值 | 说明 |
|------|-------|--------|------|
| 1 | D101 | 1 | 设备序号 |
| 2 | D102 | 1 | 序列号 |
| 3 | D104 | 11 | A排=入库口 |
| 4 | D105 | 1 | A列=1 |
| 5 | D106 | 1 | A层=1 |
| 6 | D107 | 2 | B排=2 |
| 7 | D108 | 4 | B列=4 |
| 8 | D109 | 8 | B层=8 |
| 9 | D112 | 3 | 动作类型=入库 |
| **10** | **D103** | **1** | **触发！设备开始执行** |

## 五、操作步骤

### 5.1 首次部署

```bash
# 1. 修改配置
编辑 LongShenStorageApi/appsettings.json
   → ModbusTcp.Host = PLC的IP地址
   → ModbusTcp.UseSimulator = false

# 2. 启动 API
cd LongShenStorageApi
dotnet run

# 3. 确认连接
打开浏览器 → http://localhost:5000
登录 admin/admin123 → 设备监控页面
查看"设备状态"是否正常显示
```

### 5.2 修改寄存器定义

无需改代码，直接编辑 `registers.json`：

```bash
编辑 LongShenStorageApi/registers.json
```

在 `readRegisters` 和 `writeRegisters` 数组中增删改寄存器项：
```json
{
  "address": 100,
  "name": "立库状态",
  "description": "1:空闲 2:运行 3:故障",
  "unit": ""
}
```

重启 API 生效，或通过 API 在线更新：
```
POST /api/device/config
```

### 5.3 切换模拟器/真实设备

修改 `appsettings.json`：
```json
"ModbusTcp": {
    "UseSimulator": true,    // true=模拟器, false=真实PLC
    "Host": "192.168.1.100"  // 真实PLC时填写实际IP
}
```

## 六、故障排查

| 现象 | 可能原因 | 解决方法 |
|------|---------|---------|
| 设备监控显示"未连接" | PLC 未开机 / 网络不通 | `ping 192.168.1.100` 测试连通性 |
| API 启动报错 Modbus 连接失败 | UseSimulator=false 但 PLC 未就绪 | 先设为 true 确认系统正常，再排查 PLC |
| 读取值全部为 0 | 地址偏移不对 / PLC 未运行 | 用 Modbus Poll 工具直接测试 |
| 写入不生效 | 设备不在自动模式(D102≠2) | 检查 D102 运行模式是否为 2(自动) |
| 前端显示"读取失败" | 连接断开 / 超时 | 检查网线，重启 API |
| D区地址对不上 | 不同 PLC 品牌地址映射不同 | 查阅对应 PLC 的 Modbus 地址映射表 |

### 7.2 汇川 Easy522 重要参数

| 参数 | 值 |
|------|-----|
| CPU 型号 | Easy522 |
| 编程软件 | AutoShop |
| Modbus TCP 端口 | 502 |
| 默认站号 | 1 |
| D区寄存器范围 | D0 ~ D65535 |
| 保持寄存器功能码 | 03（读取）/ 06（写入） |

### 7.3 安全注意事项

1. ⚠️ 写入操作前请确认设备处于 **自动模式**（D102=2）
2. ⚠️ 确保库位坐标（排/列/层）在有效范围内
3. ⚠️ 写入 D103=1 前必须填好所有参数
4. ⚠️ 设备运行时不要切断网络
5. ⚠️ 首次接入建议先用 Modbus Poll 工具手动验证
